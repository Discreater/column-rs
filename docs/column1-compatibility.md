# 与 `column(1)` 的差异跟踪

本文档用于持续记录 `column-rs` 与 util-linux `column(1)` 的差异，并按阶段收敛到行为一致（同时保持跨平台实现）。

## 已完成（本阶段）

- [x] 支持 `-h/--help`
- [x] 支持 `-V/--version`
- [x] 支持 `-L/--keep-empty-lines`（表格模式下）
- [x] 支持 `-s/--separator`（基础分隔符切分）
- [x] 支持 `-o/--output-separator`
- [x] 默认输出切换为列表分栏模式（保留 `-t` 表格模式）
- [x] 支持 `-c/--output-width`（基础宽度控制）
- [x] 支持 `-x/--fillrows`（基础填充顺序）
- [x] 未实现参数改为显式报错（不再当作文件路径处理）
- [x] 按显示宽度对齐（支持宽字符/组合字符宽度语义）
- [x] 支持 `-J/--json`（基础 JSON 表格输出）
- [x] 支持 `-N/--table-columns`、`-n/--table-name`（JSON 列名/表名；`-t` 下基础表头输出）
- [x] 支持 `-d/--table-noheadings`（`-t -N` 下隐藏表头）
- [x] 支持 `-H/--table-hide`（按列序号/列名隐藏列，文本/JSON 模式）

## 主要差异（待完成）

### 1. 参数覆盖不完整（高优先级）

以下 `column(1)` 参数尚未实现：

- `-C/-O/-R/-T/-W/-E/-e/-l/-m`（表格列定义/对齐/包装/隐藏/截断等）
- `-r/-i/-p`（tree 输出）

### 2. 字符宽度与对齐细节（持续校准）

- 已切换为按显示宽度计算并参与对齐。
- 与目标 `column(1)` 在更多 Unicode 边界案例上的细节行为仍需持续回归校准。

### 3. 输入解析细节（中优先级）

- 复杂分隔符、空字段、前后空白处理仍有边界差异。
- 对二进制/非 UTF-8 输入的兼容行为与 `column(1)` 不一致。

### 4. 错误信息与退出码细节（中优先级）

- 当前错误文案与 `column(1)` 不完全一致。
- 部分异常场景的退出码策略需进一步对齐。

## 建议推进顺序

1. 引入显示宽度计算，修复多语言文本对齐。
2. 逐步实现表格高级选项（列定义、对齐、截断、换行、隐藏等）。
3. 实现 JSON/tree 输出并细化错误文案与退出码兼容。

# 与 `column(1)` 的差异跟踪

本文档用于持续记录 `column-rs` 与 util-linux `column(1)` 的差异，并按阶段收敛到行为一致（同时保持跨平台实现）。

## 已完成（本阶段）

- [x] 支持 `-h/--help`
- [x] 支持 `-V/--version`
- [x] 支持 `-L/--keep-empty-lines`（表格模式下）
- [x] 支持 `-s/--separator`（基础分隔符切分）
- [x] 支持 `-o/--output-separator`
- [x] 未实现参数改为显式报错（不再当作文件路径处理）

## 主要差异（待完成）

### 1. 默认输出模式差异（高优先级）

- `column(1)` 默认是“列表分栏（columnate lists）”模式。
- 当前 `column-rs` 默认行为接近 `column -t`（表格模式）。

### 2. 参数覆盖不完整（高优先级）

以下 `column(1)` 参数尚未实现：

- `-c/--output-width`
- `-x/--fillrows`
- `-J/--json`
- `-N/-C/-O/-R/-T/-W/-H/-E/-d/-e/-l/-m`（表格列定义/对齐/包装/隐藏/截断等）
- `-r/-i/-p`（tree 输出）

### 3. 字符宽度与对齐细节（高优先级）

- 当前宽度计算使用 `chars().count()`。
- `column(1)` 在终端中按显示宽度（如 East Asian 宽字符、组合字符等）对齐。

### 4. 输入解析细节（中优先级）

- 复杂分隔符、空字段、前后空白处理仍有边界差异。
- 对二进制/非 UTF-8 输入的兼容行为与 `column(1)` 不一致。

### 5. 错误信息与退出码细节（中优先级）

- 当前错误文案与 `column(1)` 不完全一致。
- 部分异常场景的退出码策略需进一步对齐。

## 建议推进顺序

1. 实现默认“列表分栏”模式（并保留 `-t` 表格模式）。
2. 对齐输出宽度控制（`-c`）与填充顺序（`-x`）。
3. 引入显示宽度计算，修复多语言文本对齐。
4. 逐步实现表格高级选项（列定义、对齐、截断、换行、隐藏等）。
5. 最后实现 JSON/tree 输出并细化错误文案与退出码兼容。
